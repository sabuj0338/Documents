## 1. Best Practices & Security Concerns

### 1.1 Security Strategy
- **HttpOnly Cookies**: Never store JWTs or Refresh Tokens in `localStorage`. They must be handled via secure, `httpOnly`, `sameSite`, `secure` cookies set by the backend or Next.js route handlers.
- **Proxy-Level Protection**: Use `src/proxy.ts` (Next.js 16 convention) to intercept every request, ensuring unauthenticated users cannot reach private routes and authenticated users are redirected away from auth pages.
- **Validation at Every Layer**: Use Zod for client-side form validation and server-side API boundary checks.
- **RBAC**: Implement permissions at page level (Server Guards), component level (`<Can />`), API level, and navigation filtering.
- **Route Matching**: Never use `pathname.includes()` for route protection — it creates false positives like `/usersettings` matching `/users`. Use exact match + `startsWith(prefix + '/')`.

### 1.2 Recommended Development Tools
- **pnpm**: Superior disk space management and performance. Mandatory for all projects.
- **Shadcn CLI**: Generates owned UI source code — no library lock-in.
- **Turbopack**: Default dev engine in Next.js 16 for near-instant HMR.
- **OpenAPI Codegen**: Automated type generation from backend schemas.

## 4. Project Initialization

Save as `setup.sh` and run `bash setup.sh <project-name>`:

```bash
#!/bin/bash
PROJECT_NAME=$1
if [ -z "$PROJECT_NAME" ]; then
  echo "Please provide a project name. Example: bash setup.sh my-app"
  exit 1
fi

echo "🚀 Scaffolding Next.js & Shadcn UI for: $PROJECT_NAME..."

# 1. Initialize Next.js (Non-interactive)
pnpm create next-app@latest "$PROJECT_NAME" --ts --tailwind --eslint --app --src-dir --import-alias "@/*" --use-pnpm --yes

cd "$PROJECT_NAME" || exit

echo "🎨 Initializing ShadCN UI..."
pnpm dlx shadcn@latest init -d

echo "📦 Installing additional dependencies in bulk..."

# 2. Install all regular dependencies at once
pnpm add @reduxjs/toolkit react-redux \
  react-hook-form zod @hookform/resolvers \
  @tanstack/react-table axios cookies-next \
  @sentry/nextjs next-intl next-themes nextjs-toploader \
  clsx tailwind-merge lucide-react class-variance-authority \
  react-phone-number-input date-fns react-day-picker

# 3. Install all dev dependencies
pnpm add -D vitest @testing-library/react @testing-library/jest-dom jsdom playwright openapi-typescript prettier prettier-plugin-tailwindcss eslint-config-prettier

# 4. Add ShadCN components (Non-interactive)
echo "🎨 Adding ShadCN UI components..."
pnpm dlx shadcn@latest add button input label form card dialog alert-dialog \
  table dropdown-menu select textarea badge skeleton sheet separator \
  sonner avatar scroll-area tooltip popover command calendar -y

echo "✅ Setup Complete! You can now apply the boilerplate files from the guide."
```

**Why these flags:**
- `--app` → Always App Router. Never Pages Router.
- `--src-dir` → All code lives under `src/`. Keeps root clean.
- `--import-alias "@/*"` → Clean imports everywhere.
- `--use-pnpm` → Sets pnpm as the package manager.

## 9. Auth Token Strategy — Server-First Cookies

**Rule:** Never store auth tokens in `localStorage`. Do not expose refresh tokens to client JavaScript.

```typescript
export const REFRESH_TOKEN_KEY = 'refreshToken'
export const ACCESS_TOKEN_KEY = 'accessToken'

export async function getServerAccessToken() {
  const cookieStore = await cookies()
  return cookieStore.get(ACCESS_TOKEN_KEY)?.value
```

## 10. Route Protection — Proxy (Next.js 16+)

**What changed in Next.js 16:** `middleware.ts` is renamed to `proxy.ts`, and `export default function middleware` → `export function proxy`. Core functionality is identical.

### `src/proxy.ts`
```typescript
import createIntlMiddleware from 'next-intl/middleware'
import { NextRequest, NextResponse } from 'next/server'
import { routing } from './i18n/routing'

const intlMiddleware = createIntlMiddleware(routing)

const PRIVATE_ROUTE_PREFIXES = ['/dashboard', '/users', '/settings']
const AUTH_ROUTE_PREFIXES = ['/login', '/register']
const DEFAULT_LOGIN_REDIRECT = '/login'
const DEFAULT_AUTH_REDIRECT = '/dashboard'

function matchesRoutePrefix(pathname: string, prefix: string) {
  return pathname === prefix || pathname.startsWith(`${prefix}/`)
}

export function proxy(request: NextRequest) {
  const hasSession = Boolean(request.cookies.get('refreshToken')?.value)
  const { pathname } = request.nextUrl

  const isPrivateRoute = PRIVATE_ROUTE_PREFIXES.some((prefix) =>
    matchesRoutePrefix(pathname, prefix)
  )
  const isAuthRoute = AUTH_ROUTE_PREFIXES.some((prefix) =>
    matchesRoutePrefix(pathname, prefix)
  )

  if (isPrivateRoute && !hasSession) {
    const loginUrl = new URL(DEFAULT_LOGIN_REDIRECT, request.url)
    loginUrl.searchParams.set('callbackUrl', pathname)
    return NextResponse.redirect(loginUrl)
  }

  if (isAuthRoute && hasSession) {
    return NextResponse.redirect(new URL(DEFAULT_AUTH_REDIRECT, request.url))
  }

  return intlMiddleware(request)
}

export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
}
```

## 12. Server Components Strategy

**Rule:** Use Server Components by default. Add `'use client'` only when the component needs browser-only behavior.

**Use Server Components for:** Initial page data, auth/session checks, static screens, SEO pages.

**Use Client Components for:** Forms (React Hook Form), interactive tables/modals, browser APIs, Redux/RTK Query hooks.

**Use RTK Query for:** Mutations, realtime data, polling, interactive dashboards, client cache syncing.

### `src/store/slices/authSlice.ts`
```typescript
import { createSlice, PayloadAction } from '@reduxjs/toolkit'
import type { AuthUser } from '@/types/auth'

interface AuthState {
  user: AuthUser | null
  isAuthenticated: boolean
}

const initialState: AuthState = { user: null, isAuthenticated: false }

const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    setCredentials: (state, action: PayloadAction<{ user: AuthUser }>) => {
      state.user = action.payload.user
      state.isAuthenticated = true
    },
    logout: (state) => {
      state.user = null
      state.isAuthenticated = false
    },
  },
})

export const { setCredentials, logout } = authSlice.actions
export default authSlice.reducer
```

## 14. RTK Query — Base API

### `src/store/api/baseApi.ts`
```typescript
import {
  createApi, fetchBaseQuery, BaseQueryFn, FetchArgs, FetchBaseQueryError,
} from '@reduxjs/toolkit/query/react'
import { logout } from '../slices/authSlice'

const rawBaseQuery = fetchBaseQuery({ baseUrl: '/api', credentials: 'include' })

const baseQueryWithReauth: BaseQueryFn<string | FetchArgs, unknown, FetchBaseQueryError> =
  async (args, api, extraOptions) => {
    let result = await rawBaseQuery(args, api, extraOptions)
    if (result.error?.status === 401) {
      const refreshResult = await rawBaseQuery(
        { url: '/auth/refresh', method: 'POST' }, api, extraOptions
      )
      if (refreshResult.data) {
        result = await rawBaseQuery(args, api, extraOptions)
      } else {
        api.dispatch(logout())
      }
    }
    return result
  }

export const baseApi = createApi({
  reducerPath: 'api',
  baseQuery: baseQueryWithReauth,
  tagTypes: ['User', 'Auth'],
  endpoints: () => ({}),
})
```

### `src/store/api/authApi.ts`
```typescript
import { baseApi } from './baseApi'
import type { LoginPayload, AuthResponse, AuthUser } from '@/types/auth'

export const authApi = baseApi.injectEndpoints({

    }),
    getMe: builder.query<AuthUser, void>({
      query: () => '/auth/me',
      providesTags: ['Auth'],
    }),
  }),
})

export const { useLoginMutation, useGetMeQuery } = authApi
```

### `src/store/api/userApi.ts`
```typescript
import { baseApi } from './baseApi'
import type { AuthUser } from '@/types/auth'
import type { PaginatedResponse, PaginationParams } from '@/types/pagination'

export const userApi = baseApi.injectEndpoints({
  endpoints: (builder) => ({
    getUsers: builder.query<PaginatedResponse<AuthUser>, PaginationParams>({
      query: (params) => ({ url: '/users', params }),
      providesTags: ['User'],
    }),
    createUser: builder.mutation<AuthUser, Partial<AuthUser>>({
      query: (body) => ({ url: '/users', method: 'POST', body }),
      invalidatesTags: ['User'],
    }),
    updateUser: builder.mutation<AuthUser, { id: string; body: Partial<AuthUser> }>({
      query: ({ id, body }) => ({ url: `/users/${id}`, method: 'PATCH', body }),
      invalidatesTags: (result, error, { id }) => [{ type: 'User', id }],
    }),
    deleteUser: builder.mutation<void, string>({
      query: (id) => ({ url: `/users/${id}`, method: 'DELETE' }),
      invalidatesTags: ['User'],
    }),
  }),
})

export const { useGetUsersQuery, useCreateUserMutation, useUpdateUserMutation, useDeleteUserMutation } = userApi
```

```typescript
export async function requireAuth() {
  const user = await getCurrentUser()
  if (!user) redirect('/login')

export async function requirePermission(permission: string) {
  const user = await requireAuth()
  if (!hasPermission(user, permission)) notFound()
  return user
}
```

## 25. Architecture Rules

### Data Flow
```
Page → Server Component (requireAuth/requirePermission)
  → Client component (interaction boundary)
    → DynamicSearchControls + DataTable + Paginator
      → RTK Query hook → baseApi with reauth → API
  → Forms → createFormSubmitHandler() → handleFormResponse()
```

### Component Rules
- `components/ui/` → Never edit ShadCN-generated files. Custom primitives (timeline, phone-input) are your code.
- `components/common/` → No API calls. No business logic. Pure rendering.
- `components/forms/` → One form per file. Uses `createFormSubmitHandler()`.
- `components/layout/` → Shell only. No feature logic.

### State Rules
- Server-rendered data → Server Components
- Interactive server data → RTK Query (never `useState` + `useEffect`)
- Auth state → authSlice (safe metadata only)
- Tokens → server-owned cookies, never Redux
- UI state → `useState`

---

## 26. Anti-Patterns

```
❌ Making every page a client component → use Server Components by default
❌ useEffect + fetch for data → use Server Components or RTK Query
❌ localStorage for tokens → use server-owned cookies
❌ Multiple createApi() instances → inject into baseApi
❌ Raw useDispatch / useSelector → use typed hooks
❌ Hardcoded UI strings → use next-intl messages
❌ next/navigation imports → use @/i18n/navigation
❌ Editing ShadCN-generated files → use shadcn CLI
❌ Manual try/catch in every form → use createFormSubmitHandler()
❌ Building custom filter UIs per page → use DynamicSearchControls
❌ Global business types in index.d.ts → use explicit imports from src/types/*
❌ pathname.includes() for route protection → use exact/prefix matching
```

## 27. How to Add a New Feature

Example: adding a "Products" feature.

1. **Types** → `src/types/product.ts`
2. **Validation** → `src/lib/validations.ts`
3. **API** → `src/store/api/productApi.ts` (inject into baseApi)
4. **Form** → `src/components/forms/ProductForm.tsx`
5. **Columns** → define at top of page or co-located `columns.tsx`
6. **Filters** → `FieldConfig[]` + `DynamicSearchControls`
7. **Page** → `src/app/(dashboard)/products/page.tsx` with DataTable
8. **Permissions** → add to PERMISSIONS + protect with `<Can />`
9. **Nav** → add to `NAV_ITEMS` in Sidebar
10. **Messages** → add to `messages/en.json`
11. **Tests** → Vitest for permissions/validation, Playwright for CRUD
